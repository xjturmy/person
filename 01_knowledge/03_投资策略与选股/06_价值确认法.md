# 06 价值确认法 — 巴菲特 Owner Earnings（所有者收益）

> **大师**：Warren Buffett（沃伦·巴菲特，1930-）
> **方法**：Owner Earnings（所有者收益）—— 衡量股东真正能拿到手的现金
> **出处**：1986 年伯克希尔·哈撒韦致股东信
> **定位**：替代净利润的"巴菲特版"真实盈利指标

---

## 🎯 一句话核心

> "净利润是会计师算出来的，所有者收益才是股东真正能拿走的钱。"
>
> —— 巴菲特

会计净利润可以被折旧政策、并购商誉、应计项目"美化"。**Owner Earnings 还原成现金视角**，剔除维持业务必须再投入的资本支出后，才是股东真实可分配的盈利。

---

## 📐 公式（巴菲特原版）

```
Owner Earnings
  = 净利润（Net Income）
  + 折旧与摊销（D&A）
  + 其他非现金费用（Non-cash Charges）
  - 维持性资本支出（Maintenance CapEx）
  - 营运资金增加额（Δ Working Capital）
```

### 关键变量解释

| 项 | 含义 | 数据来源 |
|---|---|---|
| 净利润 | 利润表底线 | `02_盈利分析` |
| D&A | 利润表 / 现金流量表加回 | `04_现金流分析` |
| 维持性资本支出 | 维持现有产能所需 CapEx ≠ 全部资本支出 | 现金流量表（需估算） |
| Δ 营运资金 | 应收+存货 - 应付的同比变化 | 资产负债表 |

⚠️ **难点**：维持性 CapEx 没有直接科目，需估算。简化做法（实务通用）：

```
维持性 CapEx ≈ D&A      （稳态企业近似）
或
维持性 CapEx ≈ 全部 CapEx × (旧资产占比)   （成长期企业更准确）
```

---

## 🛡️ 配套筛选标准（巴菲特"15% 复合增长"门槛）

巴菲特挑选公司时，Owner Earnings 不能孤立看，要配合：

| 维度 | 门槛 | 项目数据列 |
|------|------|----------|
| Owner Earnings 增长率 | ≥ 10% / 年（10 年） | 自计算 |
| ROE（连续 10 年） | ≥ 15% | `02_盈利分析/ROE` |
| 负债率 | 长期负债 / 净利润 ≤ 5 倍 | `05_安全性分析` |
| 留存收益再投资回报 | ≥ 12% | 自计算 |
| 管理层资本配置 | 偏好回购 + 高 ROIC 收购 | 财报定性 |

## 🇨🇳 A 股适用性评估

### ✅ 适用场景

| 公司类型 | 适用性 | 理由 |
|---------|--------|------|
| 消费白马（茅台/伊利/美的） | ⭐⭐⭐⭐⭐ | 资本支出稳定，Owner Earnings 接近净利润 |
| 银行（招商银行） | ⭐⭐⭐⭐ | 巴菲特长期持有富国银行验证 |
| 周期成长（比亚迪/宁德/中际旭创） | ⭐⭐ | CapEx 大且波动，维持性 vs 扩张性难拆分 |
| 重资产周期股（中国中车） | ⭐⭐ | 同上 |
| 成长型科技 | ⭐ | 高 R&D 资本化 + 高 CapEx，公式失真 |

### ⚠️ A 股特殊调整

1. **D&A 加回需注意 R&D 资本化**：A 股科技公司常把研发支出资本化，需还原为费用化
2. **应收账款激增的公司慎用**：营运资金侵蚀严重（如部分新能源车企）
3. **少数股东损益要扣除**：A 股控股结构复杂，Owner Earnings 应取**归母净利润**

## 🧮 项目数据映射（DuckDB 落地）

```sql
-- 简化版 Owner Earnings（稳态近似法）
SELECT
    ticker,
    date,
    net_income_attributable AS ni,         -- 归母净利润
    depreciation_amortization AS da,       -- 折旧摊销
    capex_total AS capex,                  -- 资本支出
    working_capital_change AS dwc,         -- 营运资金变动
    -- 简化版：维持性 CapEx ≈ D&A
    (net_income_attributable + depreciation_amortization
     - depreciation_amortization - working_capital_change) AS owner_earnings_v1,
    -- 严格版：维持性 CapEx 单独估算
    (net_income_attributable + depreciation_amortization
     - maintenance_capex - working_capital_change) AS owner_earnings_v2
FROM cashflow JOIN profitability USING(ticker, date);
```
