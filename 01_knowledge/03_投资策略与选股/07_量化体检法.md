# 07 量化体检法 — Piotroski F-Score（9 分制）

> **大师**：Joseph Piotroski（约瑟夫·皮奥特罗斯基，1967-）
> **方法**：F-Score 9 分制 —— 用 9 项财务指标给公司"年度体检"
> **出处**：2000 年论文《Value Investing: The Use of Historical Financial Statement Information to Separate Winners from Losers》
> **定位**：低估值股票池里筛选"健康股"的过滤器

---

## 🎯 一句话核心

> "在低估值股票里，**用 9 项财务指标排除垃圾**，剩下的就是真便宜。"
>
> —— Piotroski

格雷厄姆教你"找便宜的"，但便宜里很多是**价值陷阱**（不断恶化的烂公司）。
F-Score 用 9 道财务体检题，**得分 ≥ 7 分**的低 P/B 股，长期跑赢市场年化 7.5%。

---

## 📐 9 项体检规则（每项满足 +1 分，满分 9）

### 维度 A：盈利能力（4 分）

| # | 规则 | 数据计算 | 通过条件 |
|---|------|---------|---------|
| 1 | ROA 为正 | 净利润 / 总资产 | > 0 |
| 2 | 经营现金流为正 | CFO | > 0 |
| 3 | ROA 改善 | ROA(t) - ROA(t-1) | > 0 |
| 4 | 应计质量 | CFO / 总资产 vs ROA | CFO/资产 > ROA |

> 第 4 项核心：盈利由**真实现金驱动**，而不是应计利润推高（防财务造假）

### 维度 B：财务杠杆 / 流动性（3 分）

| # | 规则 | 数据计算 | 通过条件 |
|---|------|---------|---------|
| 5 | 长期负债率下降 | LTD/资产 同比 | (t) < (t-1) |
| 6 | 流动比率改善 | 流动资产/流动负债 同比 | (t) > (t-1) |
| 7 | 无股本稀释 | 股本同比 | 未增发新股 |

### 维度 C：经营效率（2 分）

| # | 规则 | 数据计算 | 通过条件 |
|---|------|---------|---------|
| 8 | 毛利率扩张 | 毛利率同比 | (t) > (t-1) |
| 9 | 资产周转率改善 | 营收/总资产 同比 | (t) > (t-1) |

---

## 🏆 评分解读

| 得分 | 评级 | 操作建议 |
|------|------|---------|
| **8-9 分** | 强势价值股 ⭐⭐⭐⭐⭐ | 重点配置 |
| **6-7 分** | 健康公司 ⭐⭐⭐⭐ | 可纳入观察 |
| **4-5 分** | 平庸 ⭐⭐ | 谨慎 |
| **0-3 分** | 价值陷阱 🚫 | 回避 |

> Piotroski 原论文回测：在低 P/B 股池中，做多 8-9 分 + 做空 0-1 分，**年化超额收益 23%**

---

## 🇨🇳 A 股适用性评估

### ✅ 高适用度

| 行业 | 适用性 | 原因 |
|------|--------|------|
| 制造业（中国中车 / 美的 / 立讯） | ⭐⭐⭐⭐⭐ | 9 项规则的会计假设在制造业最稳定 |
| 消费品（伊利 / 五粮液） | ⭐⭐⭐⭐⭐ | 同上 |
| 工业（三美 / 三花） | ⭐⭐⭐⭐⭐ | 同上 |

### ⚠️ 需调整的行业

| 行业 | 问题 | 调整方案 |
|------|------|---------|
| 银行（招商银行） | 流动比率/总资产周转概念不适用 | 替换为：CET1 比率改善、不良率下降、净息差扩张 |
| 保险（新华保险） | 同上 + 浮存金特殊性 | 替换为：综合成本率、内含价值增长、偿付能力充足率 |
| 高研发科技（中际旭创 / 恒瑞） | R&D 资本化扭曲 ROA | 用扣除 R&D 后的"调整 ROA" |

## 🧮 项目数据映射（DuckDB 落地）

```sql
WITH yoy AS (
  SELECT
    ticker,
    date,
    -- 盈利能力 4 项
    CASE WHEN roa > 0 THEN 1 ELSE 0 END AS f1_roa_positive,
    CASE WHEN cfo > 0 THEN 1 ELSE 0 END AS f2_cfo_positive,
    CASE WHEN roa > LAG(roa) OVER (PARTITION BY ticker ORDER BY date) THEN 1 ELSE 0 END AS f3_roa_growth,
    CASE WHEN cfo/total_assets > roa THEN 1 ELSE 0 END AS f4_accrual_quality,
    -- 杠杆/流动性 3 项
    CASE WHEN ltd_ratio < LAG(ltd_ratio) OVER (PARTITION BY ticker ORDER BY date) THEN 1 ELSE 0 END AS f5_leverage,
    CASE WHEN current_ratio > LAG(current_ratio) OVER (PARTITION BY ticker ORDER BY date) THEN 1 ELSE 0 END AS f6_liquidity,
    CASE WHEN shares_outstanding <= LAG(shares_outstanding) OVER (PARTITION BY ticker ORDER BY date) THEN 1 ELSE 0 END AS f7_no_dilution,
    -- 经营效率 2 项
    CASE WHEN gross_margin > LAG(gross_margin) OVER (PARTITION BY ticker ORDER BY date) THEN 1 ELSE 0 END AS f8_gross_margin,
    CASE WHEN asset_turnover > LAG(asset_turnover) OVER (PARTITION BY ticker ORDER BY date) THEN 1 ELSE 0 END AS f9_turnover
  FROM v_company_yearly
)
SELECT
  ticker, date,
  f1_roa_positive + f2_cfo_positive + f3_roa_growth + f4_accrual_quality
  + f5_leverage + f6_liquidity + f7_no_dilution
  + f8_gross_margin + f9_turnover AS f_score
FROM yoy;
```
