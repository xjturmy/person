# 09 神奇公式法 — Joel Greenblatt Magic Formula

> **大师**：Joel Greenblatt（乔尔·格林布拉特，1957-）
> **方法**：Magic Formula（神奇公式）—— 仅用 2 个指标排序选股
> **出处**：2005 年《股市天才：投资奇才挑选超级股票的秘诀》（The Little Book That Beats the Market）
> **定位**：**最简洁的量化选股法**，2 指标，1 排序，每年换股

---

## 🎯 一句话核心

> "**好生意 + 便宜价 = 神奇公式**"
>
> "好生意"用 **ROIC（投入资本回报率）** 衡量
> "便宜价"用 **Earnings Yield（盈利收益率）** 衡量
>
> —— Greenblatt

格林布拉特用最简单的两个比率，回测 1988-2004 年美股，**年化收益 30.8%**，
是同期 S&P 500 年化收益的 2.5 倍。极简，但有效。

---

## 📐 公式（极简两步）

### 步骤 1：算两个排名

| 指标 | 计算 | 含义 |
|------|------|------|
| **ROIC** | EBIT / (净营运资金 + 净固定资产) | 衡量"好生意"——资本回报率 |
| **Earnings Yield** | EBIT / EV | 衡量"便宜"——盈利收益率（市盈率倒数升级版） |

> EV（企业价值）= 市值 + 总负债 - 现金

### 步骤 2：综合排名

```
对全市场每只股票：
  Rank_ROIC = ROIC 排名（1 = 最高）
  Rank_EY   = Earnings Yield 排名（1 = 最高）
  Combined  = Rank_ROIC + Rank_EY

按 Combined 升序排序，取前 30 只 → 等权持有 1 年 → 每年再平衡
```

## 🚫 三条排除规则（Greenblatt 原书强调）

筛选前**必须先剔除**：

1. **金融业**（银行 / 保险 / 券商）—— 财务结构不同
2. **公用事业**（水/电/燃气）—— 受管制，盈利异常
3. **市值 < 5000 万美元**（A 股可调整为 < 50 亿元）—— 流动性差

> A 股版应剔除 ST、退市风险股、北交所小市值股

## 🇨🇳 A 股适用性评估

### ✅ 高适用度

| 优势 | 说明 |
|------|------|
| 公式极简 | 只需 2 个数据：EBIT、EV、投入资本 |
| 不依赖行业判断 | 全市场量化排序，去主观化 |
| 年度调仓 | 频率低，交易成本低 |
| A 股回测有效 | 国内多家券商研究有正向超额收益 |

### ⚠️ A 股调整建议

1. **剔除清单扩大**：金融、公用事业、ST、上市 < 3 年的次新股
2. **EBIT 替代**：A 股财报无 EBIT 直接科目，用 **营业利润 + 利息支出 + 折旧摊销**
3. **持仓数量**：原版 30 只，A 股可调整为 **20-50 只**（看资金量）
4. **再平衡频率**：原版 1 年，A 股可考虑 6 个月（A 股周转更快）
5. **避免价值陷阱**：建议 **+ Piotroski F-Score ≥ 6** 双重过滤

## 🆚 与其他方法对比

| 方法 | 指标数 | 复杂度 | 主观性 |
|------|--------|--------|--------|
| **神奇公式** | **2** | ⭐ | 几乎无 |
| Piotroski F-Score | 9 | ⭐⭐ | 低 |
| 巴菲特 Owner Earnings | 4-5 项 + 定性 | ⭐⭐⭐ | 高 |
| DCF（Damodaran） | 多假设 | ⭐⭐⭐⭐⭐ | 极高 |

> 神奇公式适合**作为初筛器**，把全市场 5000 只股票筛到 30-50 只候选池。

## 🧮 项目数据映射（DuckDB 落地）

```sql
-- A 股版神奇公式
WITH base AS (
  SELECT
    ticker,
    date,
    -- EBIT 估算：营业利润 + 利息支出
    operating_profit + interest_expense AS ebit,
    -- 投入资本 = 净营运资金 + 净固定资产
    (current_assets - current_liabilities)
      + (fixed_assets_net) AS invested_capital,
    -- EV = 市值 + 总负债 - 现金
    market_cap + total_liabilities - cash AS ev
  FROM v_company_yearly
  WHERE industry NOT IN ('银行', '保险', '证券', '电力', '燃气', '水务')
    AND market_cap > 5e9                    -- 50 亿元以上
    AND NOT name LIKE 'ST%'
),
ranked AS (
  SELECT
    ticker, date,
    ebit / NULLIF(invested_capital, 0) AS roic,
    ebit / NULLIF(ev, 0) AS earnings_yield,
    RANK() OVER (PARTITION BY date ORDER BY ebit / NULLIF(invested_capital, 0) DESC) AS rank_roic,
    RANK() OVER (PARTITION BY date ORDER BY ebit / NULLIF(ev, 0) DESC) AS rank_ey
  FROM base
)
SELECT
  ticker, date, roic, earnings_yield,
  rank_roic + rank_ey AS combined_rank
FROM ranked
ORDER BY combined_rank
LIMIT 30;
```

---

## ⚠️ 局限性

1. **小盘股偏向**：原版偏好小市值，A 股小盘股波动大
2. **不看现金流**：EBIT 不等于真金白银（特别是应收账款激增的公司）
3. **无质量过滤**：单纯 ROIC + EY 高的可能是周期顶部公司（需配合 Piotroski）
4. **历史回测 ≠ 未来**：A 股近 5 年神奇公式效果较 2010 年代有所衰减
