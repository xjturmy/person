# Markdown 排版规范

统一整个知识库的 md 文件排版，提高可读性。

---

## 📐 标题层级规范

```markdown
# 一级标题 - 文件标题（仅一个）

## 二级标题 - 大章节

### 三级标题 - 小章节

#### 四级标题 - 细节（极少用）
```

**规则：**
- 整个文件只有 1 个一级标题（文件名）
- 用 ##（二级）划分主要章节
- 用 ###（三级）划分小节
- 避免使用四级以上标题

---

## 🎯 核心内容突出

**关键概念用粗体：**
```markdown
这是一个**关键概念**，需要重点理解。
```

**关键数字或公式用代码块：**
```markdown
目标价格 = `净利润 × 合理PE`
```

**重要提示用引用块：**
```markdown
> 💡 这是一个重要提示或建议
> 可以占多行，用 > 开头
```

**强调警告用红色标记：**
```markdown
⚠️ 这是一个警告，需要特别注意
❌ 这是一个反面案例
✅ 这是一个正面案例
```

---

## 📋 列表规范

**无序列表 - 用 `-` 或 `*`（统一用 `-`）：**
```markdown
- 第一点
- 第二点
  - 子项1
  - 子项2
- 第三点
```

**有序列表 - 用 `1.` `2.` 等：**
```markdown
1. 第一步
2. 第二步
   1. 细节1
   2. 细节2
3. 第三步
```

**关键行动项 - 用复选框：**
```markdown
- [ ] 完成的任务
- [x] 已完成的任务
```

---

## 📊 表格规范

**标准表格格式：**
```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 内容1 | 内容2 | 内容3 |
| 内容1 | 内容2 | 内容3 |
```

**表头对齐：**
- 左对齐：`|---|`
- 居中：`|:-:|`
- 右对齐：`|-:|`

**示例：**
```markdown
| 指标 | 数值 | 评价 |
|:----:|:----:|:----|
| PE | 15 | 合理 |
| PB | 2.0 | 偏高 |
```

---

## 💬 段落与间隔

**段落间隔：**
- 段落与段落之间**空一行**
- 不要连续多空行

**段落内不同观点分隔：**
```markdown
**优势：**
- 点1
- 点2

**劣势：**
- 点1
- 点2
```

---

## 🎨 视觉分隔线

**用于分隔主要章节：**
```markdown
---
```

**或用 emoji 分隔：**
```markdown
---

## 🎯 新章节开始
```

---

## 📝 代码块

**inline 代码（单个词汇）：**
```markdown
这里用 `代码格式` 表示字段或参数
```

**代码块（多行）：**
````markdown
```python
def 计算目标价():
    return 净利润 * PE
```
````

## 📌 常用格式组合

### 决策清单
```markdown
## 决策清单

**核心问题：**

- [ ] 问题 1？
- [ ] 问题 2？
- [ ] 问题 3？

> 📌 **关键提示**：如果无法明确回答，PASS。

**检查结果：**

| 项目 | 状态 | 备注 |
|-----|------|------|
| 项1 | ✅/⚠️ | |
| 项2 | ✅/⚠️ | |
```

### 对比分析
```markdown
## 对比分析

**优势：**
- 优点1
- 优点2

**劣势：**
- 缺点1
- 缺点2

**结论：**
> 综合判断...
```

### 步骤流程
```markdown
## 实现步骤

1. **第一步**：描述第一步做什么
   - 细节1
   - 细节2

2. **第二步**：描述第二步做什么
   - 细节1
   - 细节2

3. **第三步**：描述第三步做什么
```

## 🎯 Emoji 使用规范

| Emoji | 用途 | 示例 |
|-------|------|------|
| 📌 | 关键提示 | `📌 记住这一点` |
| 💡 | 智慧或建议 | `💡 这是一个好主意` |
| ✅ | 正面/完成 | `✅ 这是正确的` |
| ❌ | 反面/错误 | `❌ 这是错误的` |
| ⚠️ | 警告/注意 | `⚠️ 要小心这一点` |
| 🎯 | 目标/重点 | `🎯 核心目标` |
| 📊 | 数据/图表 | `📊 统计数据` |
| 📈 | 增长/提升 | `📈 上升趋势` |
| 📉 | 下降 | `📉 下降趋势` |
| 🔍 | 分析/检查 | `🔍 深入分析` |
| 💰 | 金钱/成本 | `💰 成本分析` |
| 🚀 | 开始/执行 | `🚀 立即行动` |

## ❌ 避免的做法

- ❌ 整个文件都是二级标题（标题过多）
- ❌ 段落太长（超过 5 行应该分段）
- ❌ 混用列表符号（统一用 `-`）
- ❌ 表格超过 6 列（太宽不好读）
- ❌ 没有段落分隔（视觉上压抑）
- ❌ Emoji 过度使用（每句一个太多）
- ❌ 引用块里放列表（格式会混乱）

## 📋 检查清单

在完成一个 md 文件后，用此清单自检：

- [ ] 只有 1 个一级标题？
- [ ] 标题层级合理（最多 3 级）？
- [ ] 关键概念加了粗体？
- [ ] 段落之间有空行？
- [ ] 列表格式统一？
- [ ] 表格对齐且不过宽？
- [ ] 没有连续超过 5 行的段落？
- [ ] Emoji 使用适当（不过度）？
- [ ] 没有孤立的短句？
