# 决策日志模板

> 维度 2.1 决策日志系统 — 每次买/卖/加仓必填三件事:**为什么、当时观点、5 年后预期**。
>
> 系统会在写入时自动从 [data/preson.duckdb](../../data/preson.duckdb) 抓取当时的指标快照(PE/PB/分位/ROE/...),存入 [data/decisions.duckdb](../../data/decisions.duckdb) `decisions` 表。
>
> **录入入口**:Streamlit Dashboard → Tab「📝 决策日志」(优先,3 分钟内可填完)
> **本文档作用**:Streamlit 之外的备忘 / 复盘 / 离线写法 / 审计参考

---

## ✍️ 录入字段(对应 `decisions` 表)

| 字段 | 类型 | 必填 | 说明 |
|------|------|:--:|------|
| `ticker` | str | ✅ | 6 位股票代码,如 `600519`(港股 5 位,如 `02319`)|
| `folder` | str |  | 公司目录名,如 `06_贵州茅台`(可选,Streamlit 自动填)|
| `date` | date | ✅ | 决策日期(交易日)|
| `action` | enum | ✅ | `买入` / `加仓` / `减仓` / `清仓` / `观察` |
| `weight_change` | float |  | 仓位变化百分点(+5 = 加 5%,-3 = 减 3%);`观察` 填 0 |
| `price` | float |  | 成交均价(`观察` 填当日收盘价)|
| `rationale` | str | ✅ | **为什么** — 这次行动的核心理由(2-5 句)|
| `thesis_5y` | str | ✅ | **5 年后预期** — 收入/利润/估值的具体预测,可量化更好 |
| `risks` | str | ✅ | **当时观点** 的反面 — 主要风险点(2-3 条)|
| `tags` | str |  | 逗号分隔标签:`能力圈,护城河,周期底部,...` |

**快照字段(snapshot_*)** 由系统自动写入,无需手填。

## 🧪 三件事填写要求

### 1️⃣ Rationale(为什么)

**反例 ❌**:"长期看好"、"业绩好"、"估值便宜"
**好例 ✅**:
> 茅台 PE-TTM 21x 处于 10 年 9.4% 分位,股息率 3.75% 接近 30 年期国债;批价企稳 2300+,
> Q1 直营占比 45% 创新高,经销商库存系数降至 0.6 个月。三个数据共同支持"基本面无虞、
> 估值修复"的判断,而非单一信号。

**判断标准**:
- 至少援引 **2 个独立数据点**(估值 + 基本面 / 资金 + 心理 / ...)
- 说明 **为什么是现在**(而不是 3 个月前或 3 个月后)

### 2️⃣ Thesis_5Y(5 年后预期)

**反例 ❌**:"会涨"、"看到 2000 元"
**好例 ✅**:
> 2026-2031 期间:营收 CAGR 8-10%(批价稳 → 提价空间被关税/消费税挤压一部分),
> 净利润 CAGR 9-11%,ROE 维持 30%+。5 年后 EPS 约 95-105 元,合理 PE 25-30x,
> 对应股价 2400-3100。基础情景年化收益 8-10%,加股息共 11-13%。

**判断标准**:
- 必须有 **数字**(增速 / EPS / 合理 PE / 目标价区间)
- 说明 **基础情景的核心假设**(出错时知道哪里出错)

### 3️⃣ Risks(主要风险点)

**反例 ❌**:"系统性风险"、"市场波动"
**好例 ✅**:
> 1. 消费税改革 — 若直营/经销商一体化,出厂价被迫下调 5-10%,毛利率挤压 3-5 个 pp
> 2. 批价跌破 2200 — 经销商抛货,价盘崩盘,2014 年情景重现
> 3. 接班人风险 — 张德芹 2025 年接任,3 年内是否打破"父爱算法"待观察

**判断标准**:
- 至少 **2-3 条具体的、可观察的、有触发条件的** 风险
- 不是模板话,是"如果 X 发生我会出局"的具体场景

## 🔍 自动快照内容(snapshot_json)

写入决策时,系统自动从 preson.duckdb 抓取:

| 字段 | 来源 | 用途 |
|------|------|------|
| `pe` / `pb` / `dy` | valuation 表最新 | 当时绝对估值 |
| `pe_pct_10y` / `pb_pct_10y` | 10 年窗口分位(MCP `valuation_percentile`) | 当时相对估值位置 |
| `roe` / `gm` | profitability 最新 | 当时盈利质量 |
| `rev_yoy` / `np_yoy` | growth 最新 | 当时增速 |
| `debt_ratio` | safety 最新 | 当时杠杆 |
| `fscore` | (待 2.2 接入大师评分) | Piotroski 9 项分 |

## 📅 复盘节奏(对接 2.2 决策审计)

每条决策的关键复盘节点:

- **+1 个月**:数据没大变化 → 验证 thesis 没崩
- **+6 个月**:出业绩 → 实际增速 vs `thesis_5y` 第 1 年假设差距
- **+12 个月**:能力圈矩阵更新 → 该行业胜率 +/- 调整
- **+5 年**(2031-05-03):thesis 总命中率出炉 → 错题本归档

复盘时不修改原决策,而是新建 `观察` 类型决策,在 `rationale` 里写"复盘 #N(原决策 id=X)"。

## 🛠️ 离线写法(Streamlit 不可用时)

```python
import sys; sys.path.insert(0, '.tools')
from decisions import db, snapshot

snap = snapshot.capture('600519')  # 抓最新指标
db.insert(
    ticker='600519',
    folder='06_贵州茅台',
    date='2026-05-03',
    action='加仓',
    weight_change=2.0,
    price=1685.0,
    rationale='...(见上)',
    thesis_5y='...',
    risks='...',
    tags='能力圈,周期底部,股息',
    snapshot=snap,
)
```

读取:

```python
db.list_by_ticker('600519')      # 这只票全部决策
db.list_all(limit=50)            # 最近 50 条
```
