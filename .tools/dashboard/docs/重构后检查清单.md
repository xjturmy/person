# 重构后必跑检查清单

> 每次改 dashboard 结构(尤其是导航 / 子页 / 数据加载路径)后,**按本清单跑一遍再收工**。
> 目的:拦住「页面打得开、但读不到数据」这类被自动测试漏报的故障。

---

## ⚠️ 为什么旧的检查不够(踩坑根因)

各页签的数据加载都包在 `try/except` 里,失败时渲染成 `st.error("...加载失败:...")` 而**不向上抛异常**:

```python
if page == PAGE_XXX:
    try:
        from tabs.xxx import render
        render(...)
    except Exception as exc:
        st.error(f"⚠️ XXX 加载失败:{exc}")   # ← 异常被吞在这里
```

而现有自动检查(`tests/test_app.py`、`tests/smoke/`、`bench_nav_switch.py`)**只断言
`at.exception is None`**。被 `st.error` 吞掉的失败 → `at.exception` 恒为 `None` → **100% 漏报**。

> 这就是「测试全绿,实际页面空白 / 读不到文件」反复发生的原因。
> 解决办法:除了看 exception,还要**扫描渲染树里的 `st.error` 文案**——这正是 `verify_refactor.py` 干的事。

## 🧩 维护 verify_refactor.py

新增/重命名页签或子页时,同步更新 `verify_refactor.py` 顶部的 `PAGES` 矩阵:

- `nav` 取值 = `app.py` 的 `PAGE_*` 常量字面值(带 emoji)。
- 子页 `key` = 各页 radio 的 session_state key(`navigation.py` 的 `*_SUB_TAB_KEY` / 各 tab 的 `_*_SUB_TAB_KEY`)。
- 子页取值 = radio 的 `options`(各 tab 的 `_*_SUB_IDS` / `SUB_*` 常量,**原始值**,非 format_func 后的显示文案)。
- 仍用 `st.tabs` 的页(如 🥇 黄金)一次 `run` 即渲染全部子 tab,矩阵里子页填 `[None]` 即可。
