# 🌐 共享计划 — preson-dashboard 公网部署

> 把 preson dashboard 部署成网页,朋友打开 URL 即可访问(类理杏仁体验)。
>
> **核心策略**:简化掉持仓/决策模块 → 没有隐私顾虑 → GitHub + Streamlit Cloud 免费方案。
> **代码隔离**:主仓 `preson/` 完全不动,新起子仓 `preson-dashboard/`,选择性拷贝。

---

## 🎯 目标

朋友打开一个 URL,无需安装任何东西,即可看到:
- 15 家公司的估值/盈利/成长/现金流/安全 5 维数据
- 8 大师评分矩阵(Piotroski / Buffett / Lynch / Graham 等)
- 同行雷达图、ETF 行业对标、PEG 时间曲线
- 市场周期温度计(macro 数据)

朋友**不需要**:装 Python、跑命令、下载数据库。
朋友**只需要**:浏览器(国内可访问,首次 ~5-10s)。

---

## 📋 决策记录

| 决策 | 选择 | 理由 |
|---|---|---|
| 部署形态 | **网页(服务端渲染)** | 朋友不下载数据库,不装环境;复用现有 Streamlit |
| 部署平台 | **Streamlit Community Cloud(免费)** | 月成本 0;直连 GitHub 自动构建;1GB RAM 简化后够用 |
| 代码托管 | **GitHub 私有 repo** | 免费;collaborator 邀请制 |
| 仓库策略 | **拆子仓 `preson-dashboard/`** | 主仓有大量本地工具/笔记/历史备份,边界清晰 |
| 简化模块 | **删持仓 + 决策 + Claude 终端** | 消除隐私顾虑,部署版纯财务分析工具 |
| 访问控制 | **完全公开,无密码** | 简化后无敏感数据;auth 层是多余故障点 |
| 域名 | **默认 `*.streamlit.app`** | 0 元;不用买不用备案 |
| 数据更新 | **本地 push → 5min 后线上同步** | 你跑完 update.py → cp duckdb 到子仓 → push |

## 🧩 模块封装策略

### 封装颗粒度选择

| 档 | 内容 | 工作量 | 适合场景 |
|---|---|---:|---|
| 轻量 | 不重组,仅每目录加 README | ~2h | 朋友只用不看代码 |
| **中等** ✅ **已确认** | 重组成 6 层模块 + 每模块 README + ARCHITECTURE.md | ~4.5h | 朋友能看懂结构,你长期维护 |
| 重度 | 中等 + Python package(pyproject.toml)+ sphinx 自动 docs | ~1-2 天 | 正式开源项目 |

> **2026-05-07 用户确认采用中等档**(详细工时拆解见下)。

### 中等档工时拆解(共 ~4.5h)

| 步骤 | 时长 |
|---|---:|
| 按封装结构 cp + 改名(Phase 1 部分) | 1h |
| import 路径全仓修正 + `__all__` 暴露(Phase 2 部分) | 0.5h |
| 写 6 个模块 README(每个 ~15min × 6) | 1.5h |
| 写 ARCHITECTURE.md(依赖图 + 数据流) | 0.5h |
| 写顶层 README.md(朋友友好版) | 0.5h |
| 写 methodology / data / config 三个 README | 0.5h |

### 中等档验收标准(怎么算"做完了")

1. **目录结构** — 6 层 + 8 个模块目录都建立,每个有 `__init__.py`
2. **单向依赖** — `grep -r "from pages\|import pages" core/ metrics/ scoring/ components/` 输出为空
3. **README 全覆盖** — 每模块根有 `README.md`,自动校验脚本通过
4. **架构文档** — `ARCHITECTURE.md` 含 6 层依赖 ASCII 图 + 数据流 + 5 条铁律
5. **朋友能看懂** — 不熟代码的朋友看 `README.md` + `methodology/README.md` 能在 5 分钟内理解项目做什么

### 封装的 5 条铁律

1. **单向依赖** — 上层(pages)依赖下层(core/metrics/scoring/components),下层永不反向引用上层
2. **`__init__.py` 暴露公开 API** — 每模块的 `__init__.py` 用 `__all__` 显式列出公开符号,内部辅助函数用下划线前缀私有
3. **模块内闭包** — 改 `core/data_source.py` 不必动 `pages/`(只要不破坏公开 API)
4. **配置集中** — `config/` 下所有 CSV/YAML,不散落在代码里硬编码路径
5. **数据访问只走 `core/data_source.py`** — 任何 `duckdb.connect()` 直接调用都封进 core 层,上层只看到函数
