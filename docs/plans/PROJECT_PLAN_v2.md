## 🎯 v2.0 愿景

**v1.0 评估**:数据层 / MCP / Dashboard 骨架 / 大师评分骨架全部到位 — **能查、能算、能看图**。但有一个核心缺陷:**系统是被动的,你不查它就不工作**。

**v2.0 命题**:让系统从"被动查询工具"变成"主动学习伙伴"。围绕两条主线展开:

1. **🎨 前端体验** — 把 Streamlit 4-tab 升级为投资人级仪表盘(方向 B Simply Wall St 风格已选定)
2. **📚 理财知识体系迭代** — 你的投资哲学不再只在脑子里,而是被系统记录、验证、迭代

加入两条增量主线(基于 v1.0 反思,见 [PROGRESS.md 22:30](PROGRESS.md) 4 重阻塞复盘):

3. **🤖 主动智能** — 分位/财报/异动的预警与推送
4. **🔧 工程韧性** — 测试、备份、数据广度,让系统"敢改、不丢、不挂"

## 📚 维度 2:知识层(Knowledge)

> **目标**:你的投资哲学和决策从脑子里搬到系统里,可记录、可审计、可迭代

### 🥇 2.1 决策日志系统 `[CRITICAL]`

- **核心命题**:每次买/卖/加仓必填三件事 — *为什么、当时观点、5 年后预期*。系统帮你把当时的所有数据(分位/F-Score/财报)快照存档
- **包含**:
  - DuckDB 新表 `decisions(ticker, date, action, weight_change, rationale, thesis_5y, snapshot_pe, snapshot_fscore, ...)`
  - Streamlit 新 Tab「📝 决策日志」:录入 + 列表 + 搜索
  - 模板:[01_knowledge/05_实战案例与持仓/决策日志模板.md](01_knowledge/05_实战案例与持仓/)(待建)
- **验收**:你下次买入操作时,3 分钟内能填完一条结构化记录
- **预计**:W1-W2(2026-05-10 ~ 05-23)

### 🥈 2.2 决策审计 + 能力圈追踪 `[HIGH]`

- **核心命题**:6 / 12 个月后系统自动 diff 当时观点 vs 实际表现,告诉你"哪里对、哪里错"
- **包含**:
  - `decision_audit.py`:跑全部决策记录,算每条的实际收益 + thesis 命中率
  - 能力圈矩阵:按行业 × 时间统计你的胜率(医药 70% / 白酒 50% / ...)
  - 错题本:连续 2 次同类错误 → 自动标记 + 提醒(如"你 3 次在低 PE 时入场但忽略 ROE 下滑")
- **验收**:能看到一张"我作为投资者的成长曲线"
- **预计**:W3-W4(2026-05-24 ~ 06-06)

### 🥉 2.3 投资哲学落地 `[MEDIUM]`

- **核心命题**:[01_knowledge/](01_knowledge/) 已有大量素材,但散落 — v2.0 把它结构化为"我的投资原则"
- **包含**:

## 🤖 维度 3:智能层(Intelligence)

> **目标**:系统从被动查询变主动告知

### 🥇 3.1 主动预警引擎 `[CRITICAL]`

- **核心命题**:每天扫一次,分位/财报/异动出问题立即提醒你
- **包含**:
  - **分位异常**:某指标跌破历史 5% 或突破 95% → 提醒
  - **财报暴雷**:季报当天 PE-TTM 单日跳变 > 50%(像五粮液 174.7 那次)→ 抓新闻摘要
  - **价值陷阱**:PE 低 ✅ + ROE 下滑 + 经营现金流恶化 + 负债率上升 → 输出"低估≠便宜"提示
  - **持仓异动**:单只 1 周内涨/跌 > 15% → 提示是否需要复看 thesis
  - 实现:`.tools/alerts/engine.py` + YAML 规则集(类似 8 大师评分模式)
- **验收**:五粮液那种情况发生时,系统先告诉你,不是你刷 Dashboard 才发现
- **预计**:W3-W4(2026-05-24 ~ 06-06)

### 🥈 3.2 推送通道 `[HIGH]`

- **核心命题**:预警引擎输出要送达你
- **包含**:
  - macOS 原生通知(`osascript` 或 `terminal-notifier`)— 桌面用
  - 邮件 SMTP(Gmail / QQ / 阿里云)— 移动端兜底
  - **可选**:微信机器人(企业微信 webhook,免审核)
  - 优先级:critical → 桌面 + 邮件;high → 仅邮件;info → 写日志
- **验收**:茅台 PE 跌破 5% 时你 iPhone 就能收到通知
- **预计**:W5(2026-06-07 ~ 06-13)

### 🥉 3.3 LLM 财报智能摘要 `[MEDIUM]`

- **核心命题**:每季报发布后,LLM 抽取"管理层讨论 + 关键财务变化",存到公司目录
- **包含**:
  - 抓取 02_companies/{X}/02_公司财报/ 下最新 PDF
  - 调 Claude API(prompt 缓存复用)抽取 5 类信息:营收变化原因 / 毛利率变化 / 主要风险 / 业绩指引 / 重大事项
  - 输出 `03_行业分析/季报摘要_{YYYYQX}.md`,绑定到 Dashboard 公司页
- **验收**:打开任一公司页,能看到最近 4 个季度的"老板说了什么"
- **预计**:W6-W7(2026-06-14 ~ 06-27)

## 🎯 v2.0 成功度量

完成时至少满足 4 项:

1. ✅ **5 秒判断**:打开任一公司页,5 秒内能判断"是否值得深看"(维度 1.1)
2. ✅ **决策可追溯**:任何时候能查到"3 个月前我为什么买茅台,当时数据啥样"(维度 2.1)
3. ✅ **被动 → 主动**:每月至少收到一次有效预警(分位/财报/异动)(维度 3.1)
4. ✅ **数据完整**:蜜雪有 5 年港股数据,顶部能看宏观温度计(维度 4.1)
5. ✅ **可放心改**:`pytest` 覆盖核心模块,改一处不慌(维度 4.2)
6. ✅ **可移动看**:每月 1 号 iPhone 邮箱收到 PDF 月报(维度 1.2)
7. ✅ **投资哲学有形**:1 页文档能说清"我的投资原则"(维度 2.3)

**反向度量**(失败信号):

- ❌ Dashboard 越来越重、加载 > 5 秒
- ❌ 决策日志录入 > 5 分钟一条 → 你会停止用
- ❌ 预警太多导致疲劳(每天 > 3 条)→ 噪音大于信号

## 🚫 不做(Anti-Goals)

明确**不在 v2.0 范围**,避免范围蔓延:

- ❌ **美股 / 加密扩展** — 数据源 / 税务 / 汇率全部重做,边际收益低
- ❌ **量化回测引擎** — 你不做高频/择时,买入持有 + 价值评分够用
- ❌ **多 Agent 编排** — single agent + 4 MCP 工具完全够,多 agent 是过度工程
- ❌ **扩到 30+ 公司** — 深度比广度重要,15 家都没看完不要扩
- ❌ **原生 mobile App** — PDF 月报 + 邮件简报覆盖 80% 移动需求
- ❌ **Tab 4 ttyd 嵌入** — v1.0 已确认走方案 B(VS Code 旁挂),除非用户主动切合规节点
