### 候选 ⑨ · 分层公司库扩展架构(L1/L2/L3 三层)— **🆕 用户新提需求,推荐高优先级**

> **用户原话**:"每个行业存在多个公司,现在只有 1 个,通过拉取方法从中选效率太低"

#### 问题诊断

| 现状 | 问题 |
|---|---|
| 自选库 15 家,每行业 1-2 家 | 看茅台时只能跟五粮液对比,对比单薄 |
| `peers.duckdb` 28 行(每家自选 2 个对标) | 仍局限在自选附近,看不到行业全貌 |
| 想看新行业(医药/光伏/军工)需手工拉取 | 拉一次 lixinger 全量 ~15 分钟,选筛 1 只股太重 |
| AkShare 全 A 股 spot 已用过 | 但只用于 fetch_peers 选标,没沉淀成可查询库 |

#### 推荐方案:三层金字塔架构

```text
                    ┌─────────────────────────┐
       L3 自选深度  │  15 家(理杏仁全 metric) │  周刷 / 永久
                    │  全 9 维 + 8 大师评分    │
                    └────────────┬────────────┘
                                 │ 一键升级
                    ┌────────────▼────────────┐
       L2 行业代表  │  ~200-400 家            │  月刷 / 入库
                    │  (申万二级 × Top 5-10)  │  PE/PB/ROE/F-Score lite
                    │  6 列核心,可加深跟踪    │
                    └────────────┬────────────┘
                                 │ 看上眼就拉到 L2
                    ┌────────────▼────────────┐
       L1 全市场快照│  ~5400 家全 A 股         │  周刷 / 入库
                    │  PE/PB/市值/股息/行业    │  AkShare spot 单次 ~3s
                    │  15 列轻量              │
                    └─────────────────────────┘
```

**每层维护成本**(2026-05-07 实测校准):
- L1 全市场:`stock_zh_a_spot_em` **~7-8 分钟**(中国网络分 58 batch 下载;**周末 cron 不阻塞**;~5849 行 × **23 列** / ~30MB);字段含 PE-动态/PB/总市值/流通市值/换手率/涨跌幅/振幅 等
- L2 行业代表:**首次** ~200 家 × 6 列 lixinger 抓取 ~30min;**月刷**只刷估值 ~5min
- L3 自选:维持现有 15 家全量 lixinger(已有流程不动)

> ⚠️ **重要架构决策**:L1 不能"用户点刷新立即触发"(7 分钟太久),**必须走 cron 后台预拉**。UI 显示"上次刷新:YYYY-MM-DD"。

### 候选 ⑫ · 黄金短期过热信号引擎(D2 模块扩展)— **🆕 用户新提需求**

#### 现状缺口

| D2 已有(2.4 范式投票引擎)| 缺口 |
|---|---|
| ✅ 三大范式投票(避险/抗通胀/周期)| ⚠️ **只看"长期主导身份",不看"当前买入时机"** |
| ✅ 配置比例量化(15-20% 等)| ⚠️ 不告诉你"今天该不该追高"|
| ✅ 实际利率定价(年/季节奏)| ⚠️ 周/日级动量信号缺失 |

**用户痛点**:大趋势看好 ≠ 任何时点都能进;**追高被套**是黄金投资最常见的损失模式

#### 推荐方案:6 信号过热引擎(参照 paradigm_engine 架构)

##### 信号清单(每个 3 档红绿灯)

| # | 信号 | 数据源 | 阈值速查 |
|:-:|---|---|---|
| 1 | **ETF 换手率激增** | AkShare `fund_etf_spot_em`(518880/518800 等)| 🔴 日换手 > 5% / 🟡 2-5% / 🟢 < 2% |
| 2 | **成交量爆量** | 同上(5 日均量 / 60 日均量)| 🔴 比值 > 3.0 / 🟡 2-3 / 🟢 < 2 |
| 3 | **RSI 过买** | gold.duckdb 已有金价(算 RSI-14)| 🔴 > 75 / 🟡 65-75 / 🟢 < 65 |
| 4 | **价格偏离 MA60** | gold.duckdb 已有(派生)| 🔴 > +10% / 🟡 +5~+10% / 🟢 < +5% |
| 5 | **ETF 资金流入激增** | AkShare `fund_etf_fund_info_em` ETF 份额 5 日变化 | 🔴 > +5% / 🟡 +2~+5% / 🟢 < +2% |
| 6 | **期货基差异常**(可选)| AkShare `futures_main_sina` 沪金主力 | 🔴 期货升水 > 2%(看多过度)/ 🟡 ±1% / 🟢 平水 |

##### 综合判定(6 信号 × 3 档 → 短期建议)

```
红 ≥ 3                → 🔴 暂停建仓(过热警示)
红 = 1-2 / 黄 ≥ 3     → 🟡 持有观望(局部偏热)
全绿 / 仅 1-2 黄      → 🟢 可加仓
```

**联动大趋势**(关键!):
```
大趋势看好(D2 范式 ≥2/3)+ 短期 🟢  → 加仓窗口
大趋势看好 + 短期 🟡                → 持有
大趋势看好 + 短期 🔴                → ⚠️ 暂停建仓,等过热释放
大趋势看空 + 短期 🔴                → 减仓信号
大趋势看空 + 短期 🟢                → 反弹机会(高风险)
```

- **L2 fallback 联动**:~1h(候选 ⑨ Phase 2 落地后)
- **总计 ~2-3h**

| 候选 | 解决的痛点 |
