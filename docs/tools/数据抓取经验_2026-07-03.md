## 根因

1. **短代码导致静默空数据**：`.config/companies.csv` 中部分 A 股是 `333`、`858`、`2475` 这类短代码。理杏仁 API 要求 6 位 A 股代码，短代码请求可能不报错但返回空表。
2. **BS 明细字段缺失**：Lynch/Graham 需要货币资金、应收账款等资产负债表科目，原财务模块没有正式抓 `q.bs.mc.t` / `q.bs.ar.t`。
3. **外部接口不稳定**：东方财富全 A 快照、行业映射、AkShare 部分接口会断连或返回空，需要脚本内置兜底。
4. **缓存刷新会误伤旧值**：同行脚本跳过基本面抓取时，如果直接写空字段，会覆盖 `.config/peers.csv` 的旧可用快照。

## 固化到脚本的规则

| 脚本 | 经验 |
|---|---|
| `.tools/lixinger-archiver/batch_update_fs_modules.py` | 读公司清单时统一补零；非金融 spec 加入 `q.bs.mc.t` / `q.bs.ar.t`；支持 `--only-folder` 定点补抓 |
| `.tools/db/fetch_akshare.py` | 港股用 `stock_hk_hist`；默认跳过只差当天的行情探测，强刷用 `--allow-today-probe` |
| `.tools/db/fetch_market_spot.py` | EM/Sina 行情和行业映射分层兜底；自选池行业映射优先本地缓存 |
| `.tools/db/fetch_peers.py` | 缓存模式保留旧 peer 基本面，避免 ROE/PEG 被空值覆盖 |
| `.tools/db/fetch_gold_etf_share.py` | AkShare 份额接口默认不跑；优先手填 CSV，否则用成交量代理 |
| `.tools/db/update.py` | 黄金范式/过热快照路径指向现有 `dashboard/gold/*.py` |

## 验收命令

```bash
.venv/bin/python - <<'PY'
import duckdb

con = duckdb.connect('data/preson.duckdb', read_only=True)
print('prices', con.execute("SELECT COUNT(DISTINCT ticker), COUNT(*), MAX(date) FROM prices").fetchone())
for metric in ['货币资金', '应收账款']:
    missing = con.execute("""
      SELECT COUNT(*) FROM companies c
      WHERE c.category='non_financial'
        AND NOT EXISTS (
          SELECT 1 FROM safety s
          WHERE s.ticker=c.ticker AND s.metric=? AND s.value IS NOT NULL
        )
    """, [metric]).fetchone()[0]
    print(metric, 'missing_non_financial=', missing)
con.close()

con = duckdb.connect('data/analytics.duckdb', read_only=True)
print('analytics', con.execute("SELECT (SELECT COUNT(*) FROM screener_wide), (SELECT COUNT(*) FROM company_bundle)").fetchone())
con.close()
PY
```

| 短期借款/一年内到期负债无理杏仁源 | Lynch L2 完整净现金仍不能精算 | 后续接 Tushare/AkShare 资产负债表 |
