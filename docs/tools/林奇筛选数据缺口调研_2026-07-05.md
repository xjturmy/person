# 彼得林奇筛选数据缺口调研（2026-07-05）

> 目标：检查当前 Dashboard 用彼得林奇投资法做股票筛选时，哪些数据仍缺失；并判断这些数据是否可以从理杏仁补抓。
>
> 调研口径：以 `data/preson.duckdb` 当前 100 家公司、`.tools/dashboard/masters/lynch/` 装配逻辑、`.tools/lixinger-archiver/` 抓取脚本为准。

## 一、结论摘要

当前林奇筛选的基础财务数据已经够用：PE/PB/PS、股息率、ROE、营收/净利增长、CFO/NI、负债率、历史分位等核心字段基本覆盖 100 家。

本轮补充使用 3 个并行 agent 复核后，有一个重要修正：`cash_to_market_cap` 虽然字段路径存在，但不能直接上线。当前 A 股非金融 81 家均有 `货币资金` 与 `市值(元)`，但其中约 43 家出现 `货币资金 > 资产合计` 的异常，说明 `q.bs.mc.t` 入库口径或单位存在问题。若直接接入，会误触发“资产隐蔽型”。

真正影响筛选可信度的缺口主要有四类：

| 优先级 | 缺口 | 当前状态 | 理杏仁可抓性 | 建议 |
|---|---|---|---|---|
| P0 | `cash_to_market_cap` 现金/市值 | 林奇装配层 100 家全空；现有货币资金疑似单位异常 | A 股非金融字段路径存在，但需先校验 | 先做质量保护，不直接作为筛选信号 |
| P1 | 行业毛利率基准 / 同行毛利率差 | 仅 13/100 有静态基准 | 理杏仁当前脚本未抓行业聚合毛利率 | 优先用同行库或扩静态行业基准 |
| P1 | 港股/金融 BS 明细 | 港股蜜雪、银行、保险、券商缺货币资金/流动资产等 | 当前脚本标注 HK/金融端点不支持同一字段 | 金融不进通用林奇；港股需换源或单独探针 |
| P2 | 行业故事数据：销量、提价、海外占比、市占率、渠道库存 | 未结构化，页面提示手补 | 理杏仁标准财务接口通常不能直接给 | 年报解析、行业数据库、手工摘要 |
| P2 | 机构/内部人信号 | 多数为代理或手工项 | 理杏仁当前脚本未覆盖 | 后续接 AkShare/Tushare/港交所披露 |

一句话：**理杏仁能补的是财务/估值/资产负债表类；林奇真正重视的“增长质量和故事验证”多半要靠年报、行业源或手工结构化。**

## 三、理杏仁能不能补

### 1. 可以从理杏仁补或已经补到

这些字段属于理杏仁强项：

| 字段 | 当前来源 | 状态 |
|---|---|---|
| PE/PB/PS、扣非 PE、不含商誉 PB、股息率、市值 | `/fundamental/non_financial` | 已入库 |
| ROE/ROA/毛利率/净利率/ROIC | `/fs/{category}` | 非金融覆盖好 |
| 营业收入、归母净利润、EPS、累计同比 | `/fs/{category}` | 已入库 |
| CFO、FCF、CFO/NI | `/fs/{category}` | 已入库 |
| 资产负债率、流动比率、速动比率、货币资金、应收账款、流动资产/负债 | `/fs/non_financial` | A 股非金融覆盖好 |

最明显的可利用项是：`cash_to_market_cap = 货币资金 / 市值`。当前 A 股非金融 81 家同时有货币资金和市值，但必须先加质量规则：

- 若 `货币资金 > 资产合计`，该期现金数据判为异常，不参与林奇资产隐蔽型评分。
- 页面或调试字段保留 `cash_to_market_cap_raw`、`cash_to_market_cap_warning`，方便回查。
- 该口径只是“现金/市值”，不是“净现金/市值”，未扣短期借款和一年内到期非流动负债。

### 2. 理杏仁当前脚本不适合直接补

| 字段 | 原因 | 替代 |
|---|---|---|
| 短期借款 / 一年内到期非流动负债 | 之前探针显示理杏仁字段不可用，会触发 ValidationError | Tushare/AkShare 资产负债表 |
| 港股货币资金/流动资产/流动负债 | 当前脚本明确排除 HK 这些 BS 字段 | 先单字段探针；不能直接加入生产 spec |
| 银行/保险/券商 BS 明细 | 金融端点字段体系不同，且通用林奇法本来不适用 | 金融专属模型 |
| 扣非净利润明细 / 单季扣非同比 | 理杏仁当前脚本未稳定提供 | Sina/Tushare 或年报解析 |
| 行业毛利率中位数 | 当前库没有行业聚合 profitability metric | 同行库、行业 ETF/申万成分聚合 |

### 3. 不应主要指望理杏仁补

这些是林奇“故事验证”字段，不是标准财务接口能稳定给的：

| 字段 | 用途 | 推荐来源 |
|---|---|---|
| 销量 vs 提价拆解 | 判断增长是不是靠销量扩张，而不是单纯涨价 | 年报经营数据、行业协会、公司公告 |
| 海外收入占比 | 判断增长空间和第二曲线 | 年报分部信息 |
| 市占率 / CR3 / 行业排名 | 判断是不是仍在抢份额 | 行业报告、协会、公司年报 |
| 渠道库存 / 产能利用率 | 周期股、消费股景气验证 | 年报、行业报告 |
| 单店/单模型效率 | 连锁、消费、制造新业务验证 | 公司运营数据 |
| 大股东/内部人减持、机构持仓变化 | 价格情绪和风险因子 | 交易所披露、基金季报、AkShare/Tushare |

## 四、按林奇步骤看缺口

### ① 公司分类

当前可用：行业、营收 CAGR、ROE、PE/PB、市值、负债率、净利 YoY。

主要问题：分类仍会受行业标签粗细影响。已用 A 股校正版修正一批公司，但 100 家全量仍建议继续维护“样本校准表”。

### ② 成长核查

当前可用：营收 CAGR、最新营收 YoY、净利 YoY、增长连续性派生。

主要问题：销量/提价/海外占比/市占率没有结构化。页面现在只能提示手补，无法在筛选中硬过滤。

### ③ 财务护栏

当前可用：负债率、流动比率、CFO/NI、库存周转、应收周转。

主要问题：库存/应收周转对金融和港股不完整；短期借款/一年内到期负债无稳定理杏仁字段，净现金安全边际只能简化。

### ④ PEG 估值

当前可用：88/100 可派生 `peg_lixinger`。

注意：数据库里直接名为 `PEG` 的历史字段只有蜜雪 1 家，但页面主要使用 PE + 净利润 3y CAGR 派生的理杏仁口径 PEG。缺失的 12 家不能简单理解成“没抓到”，很多是低增长、负增长或林奇 PEG 不适用。

### ⑤ 故事更新

当前可用：大部分为自动代理或手工项。

主要问题：产品突破、渠道库存、份额强化、技术替代、客户集中度、管理层行为都还没有结构化数据。该层不适合作为全量自动筛选硬条件。

### ⑥ ABCD 综合

当前可用：稳健/快速/周期三类基本能跑。

主要问题：毛利率 vs 行业、市场份额、产品价格分位、行业库存与产能等关键项仍是代理分或手工分。周期股的“产品价格 5y 分位”尤其需要行业商品/产品价格源。

## 五、建议执行顺序

### P0：先校验现金字段，再决定是否接线

1. 先检查 `货币资金` 口径异常：
   - 当前已有异常样例：贵州茅台 `货币资金` 大于 `资产合计`。
   - 该异常来自原始 `历史数据/安全性.csv`，不是 DuckDB 入库计算错误。
   - 需回到理杏仁字段 `q.bs.mc.t` 重新确认单位/字段含义，或更换现金字段来源。

2. 在林奇装配层接入受保护的 `cash_to_market_cap`：
   - A 股非金融：`safety.货币资金 / valuation.市值(元)`
   - 必须满足 `货币资金 <= 资产合计`
   - 港股：待港股现金字段补齐后再接
   - 金融：不接入通用林奇，走金融专属模型

3. 页面展示注明：
   - “现金/市值”为简化口径，未扣短债。
   - 完整净现金需要短期借款/一年内到期负债。

### P1：补行业基准

1. 扩充 `industry/gm_static.py` 到 100 家，至少覆盖非金融核心行业。
2. 或用同行库按同行公司毛利率中位数生成 `gross_margin_vs_industry_pp`。
3. 优先覆盖林奇适用公司：消费、医药、家电、制造、周期资源。

### P2：单独探针或换源

1. 港股蜜雪：探 HK 财务端点是否有现金/资产/负债字段；如果没有，走年报解析。
2. 周期股：补产品价格、库存、产能利用率，理杏仁标准财务接口不够。
3. 机构/内部人：接 AkShare/Tushare/交易所披露。

### P3：筛选规则分层

全量股票筛选建议分三层：

1. **可自动硬筛**：PE/PB/股息率、ROE、CAGR、CFO/NI、负债率、增长连续性。
2. **自动但仅作参考**：PEG、毛利率行业差、库存/应收周转、现金/市值。
3. **不可硬筛，需人工确认**：销量/提价、海外占比、市占率、渠道库存、产品突破、客户集中。

## 七、本轮优化落地（2026-07-05）

已在林奇装配层加入受保护的 `cash_to_market_cap` 接线，但默认不让异常数据参与评分或分类：

| 文件 | 变更 |
|---|---|
| `.tools/dashboard/masters/lynch/classifier.py` | 新增受保护 helper：只对 `non_financial` 使用 `货币资金 / 市值(元)`；若缺字段、金融/港股、`货币资金 > 资产合计`、或现金/市值原始值超过 80%，返回 `None` 并保留 warning |
| `.tools/dashboard/masters/lynch/classifier.py` | `现金/市值` 评分显示异常原因，不再笼统写“数据未装配” |
| `.tools/dashboard/tabs/company/block_a.py` | 概览雪花图下补回紧凑六维口径：估值 / 盈利 / 成长 / 现金流 / 安全 / 策略 |

当前验收结果：

```text
cash_to_market_cap raw: 81 家
通过质量闸门: 0 家
被屏蔽: 81 家
由现金/市值触发 asset_play: 0 家
```

结论：代码层已经避免误触发，但数据层仍需回头确认 `q.bs.mc.t` 的单位/口径。当前不应把现金/市值作为自动筛选硬条件。
