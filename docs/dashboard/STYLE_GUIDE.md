# Dashboard 视觉风格指南

> 记录时间：2026-07-07  
> 适用范围：Streamlit Dashboard（`.tools/dashboard/`）  
> 当前阶段：从“数据展示型 Dashboard”收敛为“个人投资判断副驾”

## 一句话风格

**克制、清醒、判断优先的个人投研工作台。**

这个 Dashboard 不是展示给外部看的投研大屏，也不是追求视觉冲击的产品官网。它的核心任务是帮助使用者在买、卖、持有、观察之间更冷静地做判断，并在事后能复盘当时为什么这么做。

## 当前已有风格

### 1. 工作台而非展示页

当前界面整体采用 Streamlit 宽屏布局，侧边栏负责全局导航，主区域承载分析模块。视觉重心不是装饰，而是信息读取和操作推进。

这种方向是正确的，应继续保留：

- 不做营销式首页
- 不做大幅插画、背景图或沉浸式 hero
- 不用大面积渐变和情绪化视觉
- 页面服务于投资判断，而不是服务于“看起来完整”

### 2. 低饱和语义色

当前已有一套较清晰的语义色系统，集中在 `.tools/dashboard/ui/invest_ui.py`：

- 低估 / 买入：绿色
- 合理 / 中性：黄绿或灰
- 偏贵 / 观察：琥珀色
- 风险 / 卖出：红色
- 信息 / 持有：蓝色
- 无数据 / 禁用：灰色

这套颜色克制、可读，适合投研场景。后续应继续用语义色表达状态，不用颜色做装饰。

### 3. 小圆角、轻边框、浅背景

当前组件多采用：

- 白色或浅灰背景
- 1px 浅边框
- 8px 左右小圆角
- 很轻的阴影或不用阴影
- 左侧强调线表示状态

这是适合个人工作台的视觉语言。它比厚重卡片、玻璃拟态、大阴影更适合高频阅读。

### 4. 信息密度偏高，但已经开始收敛

当前 Dashboard 已经有很多能力：

- 市场与行业
- 选股
- 公司研究
- 黄金
- 决策中心
- 多大师评分
- 同行对标
- 持仓跟踪

信息密度较高，这是投研工具不可避免的。但最近的设计方向已经从“继续加模块”转向“最终判断、操作校验、复盘闭环”，这是下一阶段最重要的风格转变。

### 5. 判断驾驶舱雏形已经存在

公司研究中已经出现类似“投资判断预览”的组件：当前动作、价格状态、核心理由、复查提示、失效条件。这说明当前风格已经开始从“评分展示”转向“行动约束”。

后续应强化它，而不是重新发明一个新页面。

## 当前风格中的主要问题

### 1. 评分感仍然强于判断感

部分页面仍然先展示分数、雷达、表格，再让使用者自行合成结论。对个人投资来说，分数只能辅助，不能成为页面的第一主语。

建议：公司研究第一屏优先展示“当前动作 + 理由 + 失效条件”，评分和图表下移。

### 2. 页面之间存在判断重复

选股、公司研究、决策中心都可能出现与“是否买卖持有”有关的信息。如果不收口，使用者会在多个页面之间来回合并判断。

建议：

- 选股页负责发现机会
- 公司研究页负责单家公司最终判断
- 决策中心负责交易前检查和持仓偏离提醒

### 3. 部分视觉元素略偏“展示型”

例如较大的 hero、综合评分块、较丰富的 tab 结构，会让公司页看起来像展示页，而不是交易前判断页。

建议：保留公司识别和关键状态，但降低 hero 的戏剧感，让“判断条”成为首屏主角。

### 4. 状态文案还可以更像投资纪律

当前不少文案是指标解释，例如“ROE 强”“PE 低位”。下一阶段需要更多纪律性表述：

- 现在不追价
- 只在价格回到区间后加仓
- 若增长失速则撤销林奇判断
- 若基本面未证伪，不因短期波动卖出

这类文案更能减少冲动操作。

## 下一阶段视觉原则

### 原则 1：首屏先给判断，再给证据

公司研究页第一屏应优先回答：

- 当前动作是什么
- 当前价格是否合适
- 为什么
- 哪些条件会让判断作废
- 下次什么时候复查

图表、评分、同行、历史数据放在判断之后，作为证据层展开。

### 原则 2：状态色只表达投资含义

颜色必须有明确含义：

| 状态 | 建议颜色 | 使用场景 |
|---|---|---|
| 买入 / 可加仓 / 低估 | 低饱和绿色 | 价格与质量同时支持行动 |
| 持有 / 信息 / 正常 | 蓝色 | 已有仓位继续跟踪 |
| 观察 / 等待 / 偏贵 | 琥珀色 | 不适合追价或证据不足 |
| 减仓 / 风险 / 卖出 | 红色 | 判断被破坏或风险显著 |
| 暂不判断 / 无数据 | 灰色 | 数据不足或方法不适用 |

不要用颜色做氛围装饰。颜色出现，就应该让使用者知道该更积极、等待、警惕，还是退出。

### 原则 3：卡片用于判断单元，不用于堆页面

可以使用卡片：

- 当前动作
- 价格状态
- 核心理由
- 失效条件
- 复查提醒
- 持仓偏离提醒

避免使用卡片：

- 包住整个页面 section
- 为装饰而套卡片
- 卡片里面再嵌套卡片
- 用过多同质卡片制造“仪表盘感”

### 原则 4：文字要短，像操作纪律

优先使用短句：

- “不追价”
- “等待回到合理区间”
- “增长失速则复查”
- “未触发卖出条件”
- “仓位偏高，暂停加仓”

少用长解释。长解释可以放在 expander 或证据区。

### 原则 5：图表退到证据层

雷达图、K 线、同行对标、历史分位仍然重要，但它们应服务于判断，而不是抢占判断。

推荐层级：

1. 判断条
2. 核心证据摘要
3. 方法内判断
4. 图表与表格
5. 原始数据与长解释

## 推荐页面结构

### 公司研究页

首屏结构建议：

1. 公司选择器与视角切换：当前持仓 / 观察池 / 全部公司
2. 投资判断条：动作、价格、方法、复查
3. 三条核心理由
4. 失效条件
5. 方法内判断入口：概览 / 林奇 / 格雷厄姆 / 芒格
6. 图表、评分、同行、财务细节

公司研究页是最终判断中枢，不再只是数据详情页。

### 选股页

选股页只回答：

- 哪些公司值得看
- 为什么进入观察池
- 哪些公司应从观察池删除

选股页不直接承担买卖决策。

### 决策中心

决策中心只回答：

- 当前持仓是否偏离目标
- 是否触发复查
- 买卖前是否违反原计划
- 是否需要记录一条决策

它不应重复公司研究页的大量分析。

## 组件规范

### 投资判断条

用途：公司页首屏主组件。

应包含：

- 当前动作：买入 / 观察 / 持有 / 减仓 / 卖出 / 暂不判断
- 价格状态：便宜 / 合理 / 偏贵 / 不适合判断
- 主方法：林奇 / 格雷厄姆 / 芒格 / 保险专属 / 暂无主方法
- 下次复查：季报 / 年报 / 价格触发 / 指标触发

视觉：

- 白底
- 1px 边框
- 左侧 4px 状态色
- 8px 圆角
- 高度紧凑，不做大 hero

### 核心理由

用途：解释当前动作。

规则：

- 最多 3 条
- 每条一句话
- 优先写结论，不写指标堆砌
- 需要指标时只放一个关键数

### 失效条件

用途：防止事后改口。

规则：

- 必须明确
- 尽量可观察
- 不超过 3 条
- 用警示色，但不要过度渲染

示例：

- 同店增长连续两个季度转负
- 毛利率趋势性下滑且无法解释
- 估值进入高分位但增长没有同步提升

### 状态 Badge

用途：表达状态，不表达装饰。

建议尺寸：

- 字号 12px
- pill 圆角
- 浅底色 + 深文字 + 小圆点

不建议：

- 大量 emoji 叠加
- 高饱和背景
- 同一行超过 4 个 badge

## 字体与密度

### 字号建议

| 元素 | 字号 |
|---|---:|
| 页面主标题 | 22-26px |
| 区块标题 | 18-20px |
| 判断动作 | 18-22px |
| 指标值 | 20-24px |
| 正文 | 13-15px |
| 辅助说明 | 12-13px |
| 表格 | 12-13px |

不要用随视口缩放的字体。不要用负字距。

### 密度建议

这是个人高频工具，应偏紧凑：

- 组件间距小而清楚
- 表格高度可滚动
- 长说明默认收起
- 首屏不超过一个核心判断区 + 一组理由

## 颜色基准

延续当前语义色：

```text
Text       #111827
Muted      #6B7280
Border     #E5E7EB
Surface    #FFFFFF
Soft       #F9FAFB
Accent     #1F6FEB
Green      #16A34A
Blue       #2563EB
Amber      #D97706
Red        #DC2626
Gray       #6B7280
```

后续新增颜色应优先从这组派生，避免页面变成多套视觉语言。

## 不建议继续强化的方向

- 新增更多顶级导航
- 新增展示型大屏
- 新增纯装饰视觉
- 为每个模块设计不同颜色体系
- 让评分成为公司页主视觉
- 在一个页面同时突出太多“主结论”
- 把持仓页做成公司研究页的复制版

## 下一步落地建议

第一期优先做：

1. 公司研究页首屏改为“判断条优先”
2. 公司选择器增加：当前持仓 / 观察池 / 全部公司
3. 将综合评分和雷达图下移为证据层
4. 决策中心只保留操作检查和持仓偏离提醒
5. 选股页只保留发现与观察池管理，不输出最终买卖结论

## 风格验收标准

一次 UI 改动是否符合当前风格，用这些问题判断：

- 第一屏是否能看出现在该不该动？
- 是否减少了一个需要使用者脑内合成的判断？
- 是否把分数放回辅助位置？
- 是否暴露了失效条件？
- 是否避免新增一个重复页面？
- 是否能阻止一次追涨、恐慌卖出或无计划加仓？

如果答案多数为否，这次改动就不符合当前阶段的 Dashboard 风格。
