# 公司研究概览页候选版本

> 生成时间：2026-07-07  
> 目标：为“公司研究 → 概览”选择下一版可视化方向。  
> 说明：这些是静态候选稿，不影响正式 Dashboard 代码。

## 候选版本

| 版本 | 文件 | 图像 | 方向 |
|---|---|---|---|
| A | `vA_minimal_decision.html` | `vA_minimal_decision.png` | 极简判断优先 |
| B | `vB_evidence_grid.html` | `vB_evidence_grid.png` | 判断 + 证据网格 |
| C | `vC_holding_action.html` | `vC_holding_action.png` | 持仓操作约束 |

## A 极简判断优先

适合做默认概览的最克制版本。它把首屏压缩到公司识别、当前判断、三条纪律和很少的证据标签。

优点：最清楚、最不分心、最符合“先判断再证据”。

不足：证据感偏弱，用户如果想研究原因，需要继续往下看。

## B 判断 + 证据网格

适合做正式概览页主方案。它保留判断条，同时把综合评分、强项、待验证、失效条件、下次看什么放在可扫描网格里。

优点：判断与证据平衡，最像成熟的投研工作台。

不足：信息量比 A 更高，需要避免后续继续堆内容。

## C 持仓操作约束

适合“当前持仓”范围下使用。它先回答仓位和操作纪律：持有、不加仓、核查区间、记仓位上限。

优点：最能阻止冲动操作，适合真实持仓复查。

不足：对非持仓公司不自然，不适合作为全部公司的默认概览。

## 建议

推荐采用组合策略：

- 默认概览：B
- 当前持仓范围：C
- 紧凑模式或移动端：A

如果只能先落一个版本，建议先落 B，再把 C 的“不要做 / 可以做 / 必须记”作为当前持仓时的增强区块。
