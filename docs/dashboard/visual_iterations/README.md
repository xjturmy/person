# 公司研究首屏视觉迭代

> 生成时间：2026-07-07  
> 目标：为公司研究页的“持仓优先 + 判断优先”首屏保存多个静态候选版本，供人工横向比较。

## 文件

| 版本 | 文件 | 定位 |
|---|---|---|
| V1 | `company_research_v1_minimal.html` | 极简判断条 |
| V2 | `company_research_v2_decision_grid.html` | 判断 + 证据网格 |
| V3 | `company_research_v3_holdings_first.html` | 持仓优先 |

## V1 极简判断条

设计目标：把页面压缩成“现在该不该动”的纪律卡。

适合场景：

- 日常快速打开看一眼
- 防止冲动交易
- 想让判断条成为绝对主角

优点：

- 信息最少，注意力最集中
- 很符合“个人投资副驾”的方向
- 后续接入正式 Dashboard 成本低

缺点：

- 证据感较弱
- 对新进入观察池的公司解释不够充分

推荐程度：高。适合作为默认首屏骨架。

## V2 判断 + 证据网格

设计目标：在不回到大仪表盘的前提下，保留核心证据。

适合场景：

- 研究一家公司时希望同时看到判断和证据
- 对持仓和观察池都比较均衡
- 希望保留评分、强弱项等辅助信息

优点：

- 信息完整度更高
- 判断、理由、失效条件、评分证据层级清楚
- 更适合作为“公司研究概览页”的完整方案

缺点：

- 首屏信息略多
- 需要控制右侧证据栏不要继续膨胀

推荐程度：最高。适合作为正式改版主方案。

## V3 持仓优先

设计目标：打开持仓公司时，先看仓位纪律和操作约束，再看研究证据。

适合场景：

- 当前持仓管理
- 加仓、减仓、卖出前检查
- 决策中心与公司研究联动

优点：

- 操作约束最强
- 能直接回答“现在要不要动仓位”
- 适合真实持仓的日常复查

缺点：

- 对非持仓公司不够自然
- 需要接入真实仓位、目标仓位、偏离度后才完整

推荐程度：中高。适合做“当前持仓”范围下的专属首屏。

## 建议

正式 Dashboard 可以采用组合策略：

- 默认公司研究：用 V2。
- 当前持仓范围：吸收 V3 的仓位纪律卡。
- 移动端或紧凑模式：使用 V1。

不要把三个版本都做成独立页面。它们应当成为同一公司研究页在不同视角下的布局策略。
