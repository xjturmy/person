## 一、估值口径(2026-05-06 立)

### 🎯 核心规则:**PE / PB / PEG 一律使用扣非口径**

| 指标 | 优先字段(主用)| 后备字段(fallback)| 数据源 |
|---|---|---|---|
| **PE-TTM** | `PE-TTM(扣非)` | `PE-TTM`(GAAP)| 理杏仁 valuation 表 |
| **PB** | `PB(不含商誉)` | `PB`(GAAP)| 理杏仁 valuation 表 |
| **PEG** | 基于 **扣非净利润 TTM YoY** | 基于归母净利润 TTM YoY | peg_curve.py 重算 |

### 为什么用扣非?

- **扣非 = 扣除非经常性损益**(资产处置 / 政府补助 / 公允价值变动 / 一次性税收返还等)
- **更代表持续经营盈利能力** — 林奇五步分析、ABCD 评分都看的是"未来 5-10 年能稳定赚多少",非经常项是噪声
- **避免周期 / 一次性事件干扰估值** — 例:某年卖掉子公司大赚一笔,GAAP 净利润虚高 → PE 显得很低,但不可持续

### 为什么 PB 用"不含商誉"?

- 商誉是并购溢价,非经营性可变现净资产
- 减值风险大(每年商誉减值测试,一次减值就抹掉账面价值)
- 林奇 / 格雷厄姆原文都偏向"有形净资产"口径(类似巴菲特的 owner's earnings 思想)
- "PB(不含商誉)" 是这个口径的本地等价物

### 代码实现规范

所有读 valuation 表的代码使用统一辅助函数:

```python
def _val_metric(con, ticker: str, primary: str, fallback: str) -> float | None:
    """主用扣非口径,缺则 fallback 到 GAAP。"""
    v = _latest_value(con, "valuation", ticker, primary)
    if v is None:
        v = _latest_value(con, "valuation", ticker, fallback)
    return v

# 调用模板
pe = _val_metric(con, ticker, "PE-TTM(扣非)", "PE-TTM")
pb = _val_metric(con, ticker, "PB(不含商誉)", "PB")
```

**规则**:
- ✅ 默认主用扣非字段
- ✅ 缺数据时无声 fallback(避免大量公司 None)
- ✅ UI 显示时若使用 fallback,在 tooltip 标注"GAAP 口径(扣非数据缺)"

## 二、抓取脚本 metrics 列表必含项

`.tools/lixinger-archiver/batch_update_recent_wide.py` 抓取的 metrics 必须包含:

| 类别 | 必含字段 |
|---|---|
| **估值(扣非主)** | `pe_ttm_2`(PE-TTM 扣非)+ `pb_wo_gw`(PB 不含商誉)+ `pe_ttm`(GAAP 备用)+ `pb`(GAAP 备用)|
| **盈利** | ROE / ROE(扣非)/ 净利率 / 毛利率 |
| **成长** | 营收 + 归母净利润 + **扣非归母净利润** |
| **股息** | 股息率(`dyr`)|
| **分位** | y10 / y5 滚动分位点(扣非主用)|

理杏仁 API metricsList 字段名对照:
- `pe_ttm` = PE-TTM(GAAP)
- `pe_ttm_2` = PE-TTM(扣非)
- `pb` = PB(含商誉)
- `pb_wo_gw` = PB(不含商誉)
- `dyr` = 股息率

## 三、PEG 计算口径

**主算式(扣非)**:
```
PEG = PE-TTM(扣非) ÷ (扣非净利润 TTM YoY × 100)
```

**备用算式**(扣非数据缺时):
```
PEG = PE-TTM(GAAP) ÷ (归母净利润 TTM YoY × 100)
```

**禁用**:用营收 CAGR 当分母 — 这是错误口径,会系统性低估科技 / 周期股 PEG。
