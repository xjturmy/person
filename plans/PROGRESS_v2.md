### 1.1 产出明细

**核心文件**:

- [.tools/dashboard/score_card.py](.tools/dashboard/score_card.py) — 5 维 0-100 评分函数 + 综合分 + 雷达数据装配,可独立 CLI 验证
- [.tools/dashboard/app.py](.tools/dashboard/app.py) `tab_home` — 雷达图 + 5 卡片 + Piotroski 子项展开
- [.tools/dashboard/REDESIGN_TODO.md](.tools/dashboard/REDESIGN_TODO.md) — checklist 全部勾选

**5 维评分映射**:

| 维度 | 输入 | 区间映射 | 权重 |
|------|------|---------|----:|
| 估值 | PE-TTM 全周期分位(0-1) | 100·(1-pct);分位越低分越高 | 25% |
| 盈利 | 最新 ROE | 0% → 0,25%+ → 100 线性 | 25% |
| 成长 | 营收累积同比 | -10% → 0,+25% → 100 线性 | 15% |
| 现金流 | CFO/净利润 | 0 → 30,1.0 → 90,≥1.2 → 100 | 15% |
| 安全 | 资产负债率(银行/保险跳过) | 30% → 100,70% → 0 反向 | 20% |

**综合分阈值**:75+ 🟢 / 60+ 🟡 / 45+ 🟠 / <45 🔴

| 美的 000333 | 🟠 58 | 56 | 73 | 36 | 100 | 25 |
| 三花 002050 | 🟡 63 | 22 | 64 | 60 | 100 | 87 |

**AppTest 验证**:6 tabs 全渲染 / 切换 3 家公司 0 exception / 唯一 warning 是 ttyd 未运行(无关本任务)。

- 招商银行/新华保险 安全维度按"不适用"返回 N/A,综合分用 50 中性占位 — 待 step-04 P3 金融业字典解锁后改专用规则

### 1.2 月报雏形产出明细(W0 提前)

**核心设计**:不直接出 PDF,而是 **Claude-prompt 消费模式** — 数据收集器收齐输入,Claude 用预定义 prompt 写月报。这样格式可灵活,不绑死 PDF 库。

- [.tools/portfolio/monthly_review.py](.tools/portfolio/monthly_review.py) — 数据收集器:持仓 / 评分 / 估值 / 偏离 / 候选 → markdown
- [.claude/prompts/monthly_review.md](.claude/prompts/monthly_review.md) — Claude prompt 模板,消费 monthly_review.py 输出
- 输出位置:`.temp/monthly_review_YYYY-MM.md`

**剩余工作**(留 W3):
- PDF 渲染(`weasyprint` 或 `markdown2pdf`)
- 邮件发送(SMTP)
- LaunchAgent 月度 cron(每月 1 号 09:00)
