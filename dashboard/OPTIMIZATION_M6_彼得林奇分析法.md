## 🎯 优化目标

**新增独立 Tab `🌱 林奇分析法`**,把彼得林奇成长投资法做成与价值投资法 5 步并列的**结构化分析工作台**,每家公司一键跑五步、自动给结论。

为什么独立成模块而非塞进单公司详情:
- **故事驱动**:Lynch 的核心是"先分类,再走差异化路径",6 类公司用 6 套估值口径 — 单 Tab 装不下
- **流程化**:用户期望"按步骤填空 → 自动出结论"的引导式体验,不是看一堆图表自己拼
- **决策日志直接落地**:跑完 5 步可一键写到 `02_companies/{N}_{name}/05_投资决策/林奇五步分析_{date}.md`(对照已有的[新华保险格雷厄姆五步分析](../../02_companies/01_新华保险/05_投资决策/02_格雷厄姆投资法_新华保险五步分析.md)模板)

## 🆕 5 步映射(对照价值投资 5 步重设计)

| 步 | 价值投资法(格雷厄姆) | 成长投资法(林奇) · 本模块 | 核心问题 |
|:-:|---|---|---|
| 1 | 商业模式理解 | **公司分类(六类判断)+ 故事脚本** | 这家是哪一类?故事是什么? |
| 2 | 盈利能力与成长性 | **成长核查(增速质量 + 持续性)** | 增长是真还是假?能持续多久? |
| 3 | 财务健康 | **财务护栏(高增长不烧钱)** | 负债率 / 现金流 / 库存周转过关吗? |
| 4 | 估值 | **PEG 估值(成长合理性)** | 当前价格相对增速是贵是便宜? |
| 5 | 深度审视 | **故事更新(每季 ping 兑现度)** | 故事还在轨吗?何时该卖? |

## ⚠️ 风险与注意

| 风险 | 处理 |
|---|---|
| **PEG 不适用亏损公司** | UI 上明确标"不适用,改 PSG/EV-EBITDA",不强出数字 |
| **林奇六类口径互不通用** | 第 3/4 步阈值随类型变 — 测试覆盖至少 3 类公司(快速/稳健/周期) |
| **故事脚本是主观字段** | session_state + 写入文件,下次进同一公司能拉回上次的故事 |
| **15 家公司中可能没有典型快速增长型** | 比亚迪/宁德/三花智控接近,茅台/招行不适用 — 文档说明优先验证哪些公司 |
| **写入决策日志和 portfolio.yaml 重叠** | 林奇决策日志走 `02_companies/.../05_投资决策/` md(细节),portfolio.yaml 仅记 ticker+权重(摘要) |
| **lynch_classifier.py 已有 773 行,需复用不重写** | 调用其 `_classify_lynch(ticker)` 即可,新模块只做 UI + 流程编排 |

## 💡 设计亮点

1. **流程化引导**:5 步按钮顺序排列,每步顶部"📖 此步回答:..." 强引导
2. **类型驱动阈值**:第 1 步选完类型后,3/4 步阈值自动适配(快速 vs 稳健 vs 周期)
3. **林奇原话嵌入**:在每步关键阈值旁引用林奇原话(强化纪律感,不抽象)
4. **故事-卖出闭环**:第 1 步填故事 → 第 5 步打分,自动生成卖出条件
5. **决策落地**:"📤 导出 md"直接写到公司文件夹,与人工分析格式一致
6. **离线可测**:`lynch_steps.py` 纯逻辑无 streamlit 依赖,headless test 跑得通
